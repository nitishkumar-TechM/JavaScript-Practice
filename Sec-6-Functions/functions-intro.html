Function Declarations vs Function Expression

Function Declaration

A function declaration defines a named function that can be called later in the code.

        function greet() {
            console.log("Hello, World!");
        }

When we defining the function using function declaration syntax, so we can
call the function before it is defined in the code. But this is not possible with function expressions.

The reason for this is because when our JavaScript engine executes this code,
it hoists/moves the function declaration to the top of the scope,
allowing us to call the function before it is defined.

Hoisting

Hoisting is the process of moving declarations to the top of their scope before the code is executed.
This means that function declarations are available throughout their scope, even before they are defined in the code.
And this done automatically by the JavaScript engine that is executing the code.

Hoist (meaning) : To raise, to lift
